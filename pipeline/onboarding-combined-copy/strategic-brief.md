# Strategic Brief — Onboarding & Combined Copy

## What We're Building
Two popup UX improvements shipped together in one lane: (1) a **first-run
key walkthrough** that guides a brand-new user from install to their first
lookup when no API keys are set, and (2) a **combined "Copy weather & tide
together" button** that copies a single block matching the SnowRaven desktop
app byte-for-byte.

## Why Now
The extension is live on both stores at v1.2.0 and the core lookup is solid,
so the remaining friction is at the two ends of the experience. New users hit
a bare "missing-keys" notice with no path forward — the worst possible first
impression for an otherwise polished tool — while returning power users who
want both blocks in one comment have to copy twice and stitch them by hand,
when SnowRaven already produces the exact combined output they want. Both are
small, self-contained, and close gaps the founding brief already named
(one-time setup; faithful-to-SnowRaven output).

## The User Problem
A first-time birder installs the extension, opens the popup on a checklist,
and sees only a notice that keys are missing — with no explanation of which
keys, where to get them (both are free), or what to do next; some will give
up before their first successful copy. Separately, a birder who wants both
weather and tide in their comment must copy the weather block, copy the tide
block, and assemble them with the right spacing and a single attribution
line — work SnowRaven's desktop app does for them in one button, and that the
extension currently can't.

## Success Criteria
- A new user with no keys sees a clear, guided path — what the two free keys
  are (eBird + OpenWeather), where to get them, and how to open Settings —
  instead of a dead-end notice, and reaches their first successful lookup
  without leaving the popup confused.
- When both weather and tide are present, a single "Copy weather & tide
  together" button appears alongside the existing per-block copy buttons.
- Clicking it places on the clipboard a block that is **byte-for-byte
  identical** to SnowRaven desktop's `buildCombined()` output, verified by a
  golden test asserting the exact string with `.toBe`.
- The combined copy is **manual only** — the popup keeps auto-copying weather
  alone on open, exactly as today; nothing about existing behavior changes.
- Both additions hold WCAG 2.1 AA: keyboard-reachable, labelled, and async
  state changes (copy confirmation, walkthrough steps) announced through the
  shared polite live region.
- No new permissions, no backend, no measurable bundle growth beyond the UI
  these two additions require.

## Scope
- **First-run walkthrough:** when the popup opens with no keys set, replace
  today's bare missing-keys notice with onboarding that names the two free
  keys (eBird + OpenWeather), points to where to get them, and routes the
  user to Settings to paste them. Mini-specific; no SnowRaven desktop parity
  to honor here.
- **Combined copy button:** an additional "Copy weather & tide together"
  button in the popup, shown only when both weather and tide are present,
  alongside (not replacing) the existing per-block copy buttons.
- **Combined output, byte-for-byte to SnowRaven `buildCombined()`:** the
  weather block with its standalone attribution line stripped, a blank line,
  the tide body, a blank line, and a single combined attribution
  `Weather and tide generated by <a href="https://github.com/dtgibson/snowraven">SnowRaven</a>`.
  Locked with a golden `.toBe` test.
- Both additions meet WCAG 2.1 AA, announcing through the shared live region.

## Out of Scope
- **Auto-copying the combined block.** SnowRaven never auto-copies the
  combined output, and neither will Mini — it stays a deliberate user action.
  The popup's existing weather-only auto-copy on open is unchanged.
- **The duplicate-block guard** (skipping auto-copy when the comment already
  contains a weather block) — separate roadmap item, not this lane.
- **Tide without an OpenWeather key** (showing tide on its own when only
  OpenWeather is missing) — separate roadmap item, not this lane.
- **Any combined copy on the marketing site** — site is untouched.
- No new weather or tide logic, no new permissions, no backend.

## Key Decisions
- **Parity is the hard contract for the combined copy.** The output must
  match SnowRaven desktop's `buildCombined()` exactly and is golden-tested
  with `.toBe`; this is a port-and-lock, not a re-implementation. (CLAUDE.md)
- **Combined copy is manual-only and conditional.** It appears only when both
  blocks are present and is never auto-fired — mirroring SnowRaven and
  preserving the "weather copies on click; tide is yours to add" founding
  decision.
- **The combined button is additive.** It sits alongside the existing
  per-block copy buttons; no existing copy behavior is removed or changed.
- **The walkthrough is Mini-specific.** There is no desktop equivalent to
  port; it serves the founding brief's "setup is a one-time paste of two
  keys" promise and the same birder, met at first run.
- **Both items ship in one lane** even though the roadmap lists them under
  two "Up Next" rows — both are popup UX, small, and independent of the
  duplicate-block guard and tide-without-OpenWeather work parked alongside
  them on the roadmap.
- **No drift from the founding brief.** Both additions serve the same birder,
  reuse SnowRaven's output, add no permissions, and keep the popup-only,
  on-device, minimal-footprint posture intact.
